Стандартная реализация HashMap в которой когда количество елементов больше критического 
( threshold ) то мы увиличиваем размер если он не максимален (ибо только так можно уменьшить количесвто колизий) 

transfer просто переопределяет старый массив ссылок на новый

AdaptSize проверяет или надо менять размер и если надо меняет 

addNodeToTable добавляет нужную пару к новому списку 

Table списов ссыло

Node елемент где записана информация про данную пару.

IHashMap интерфейс где описаны главные открытые функйции

loadFactor это файтор что показывает отптимальное число памяти для предатвращения полизий. 
поскольку чем больше число тем больше шанс, он постименно уменьшаеться.
